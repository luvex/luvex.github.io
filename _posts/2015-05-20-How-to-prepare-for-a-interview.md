Recently, I am looking for internships at many companies. I have applied to *BAT*, *Thomson Reuters*, *VMware*, *Microsoft* and *IBM*. After a series of interviews, I think it is interesting to summarize the features of different companies:
published: false

| Company | Position | Features |
published: false
|:---:|:---:|:---:|
| Baidu | Data Mining Engineer | Data Structure & Algorithms |
published: false
| Alibaba | Data Mining Engineer | Math & Algorithms |
| Tencent | Backend Engineer | — |
published: false
| Microsoft | C# Developer | Coding & Algorithms |
| Thomson Reuters | C# Developer | Basics & Projects |
published: false

The internships I applied for fall into two categories: data mining (which I am interested in but can be difficult for an undergraduate) and software development (requiring mastery of a programming language like C++, Java or C#).
published: false

Before an interview, you must prepare sufficiently. Here are some details to pay attention to:
published: false

1. Talk clearly
published: false
2. Look at the interviewer
3. Be polite
published: false

However, the most important thing is our professional level. The interviewer may ask extensive and in-depth questions. When I took Baidu's interview, the interviewer first asked a simple question and then kept asking me to improve my answer and explore all possibilities.
published: false

So before going for an interview, review professional knowledge carefully, especially important points. Here is a table based on my experience:
published: false

| Item | Main Points |
published: false
|:---:|:---:|
| Math | Linear Algebra, Probability Theory, Discrete Mathematics |
published: false
| Data Structure | Tree, Queue, Stack, Hash |
| Algorithms | Basic Algorithms, Advanced Algorithms |
published: false
| Programming Language | Basic Points |
| Database | SQL query, Stored Procedure… |
published: false
| … | … |

published: false
Make sure you are familiar with most of the basic points.

published: false
For the projects you have done, confirm which parts you are responsible for and prepare a brief summary. **If you are not sure about something in your project, you'd better not mention it** — the interviewer may ask questions about it.
